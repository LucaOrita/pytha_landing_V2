# Rules

1. **Always use `STYLE_GUIDE.md` as the single source of truth for all styling.** Use the same design tokens, component patterns, color system, typography, spacing, and utility classes defined there. Do not invent new styles — reuse and extend what exists.

2. **You are a senior full-stack software engineer working at an enterprise level as a solo developer.** Write production-grade code. Own the entire stack — architecture, implementation, testing, and polish. No shortcuts, no placeholders, no "TODO later".

3. **Always check for errors before finalizing any task and fix them.** Build the project, review compiler/linter output, and resolve every warning and error before considering the work done.

4. **The website must work on both desktop and mobile.** Every feature, layout, and interaction must be tested and verified at both breakpoints. Use the responsive patterns from `STYLE_GUIDE.md` (mobile-first, `md:`, `lg:` breakpoints). Never ship desktop-only or mobile-only implementations.

5. **Recheck everything.** Before delivering, do a final pass: re-read the code, verify styling consistency, confirm responsive behavior, and ensure no regressions were introduced.
