# PARTEA I: CONTEXTUL BUSINESS

## 1. Ce este Pytha

**Pytha 3D CAD** este un software german de proiectare 3D dedicat industriei mobilierului. Produs de **Pytha Lab GmbH** (Germania), activ pe piață de peste 25 de ani, cu peste 500 de firme utilizatoare în Europa. Este vândut în România prin distribuitorul local **Adline Industries** sub brandul **Pytha 3D CAD Romania**, pe site-ul **pytha3d.ro**.

**Preț:** ~20.000 EUR per licență (high-ticket B2B, ciclu de vânzare lung).

**Ce face Pytha în termeni simpli:** Transformă procesul de proiectare a mobilierului dintr-un flux manual (schițe pe hârtie / SketchUp / AutoCAD + software CNC separat + calcul manual materiale) într-un ecosistem digital unic care face totul: design 3D, documentație tehnică, listă piese, optimizare tăiere panouri, generare cod CNC, și randare fotorealistică. Totul într-un singur software, fără export/import între programe diferite.

**Propunere de valoare principală:**
"PYTHA unifică designul 3D, documentația tehnică și generarea codului CNC într-un singur ecosistem. Elimină erorile manuale, accelerează producția și economisește până la 40% din timpul angajaților tăi. Fără plug-in-uri. Fără exporturi inutile. Doar eficiență pură."

## 2. Modulele Pytha (structura de produs)

| Modul | Ce face | Pentru cine |
|-------|---------|-------------|
| **Pytha Design** | Modelare 3D parametrică. Redimensionare instantanee a mobilierului, biblioteci de corpuri, planuri de debitare automate, documentație tehnică. | Toți utilizatorii — modulul de bază |
| **Pytha Workshop** | Generare automată metode de construcție (dibluri, șuruburi, cam & dowel). Edge banding, router paths. Generare cod CNC direct — elimină nevoia de software CAM separat (Alphacam, etc.). | Fabrici cu mașini CNC |
| **Pytha Rendering** | Randare fotorealistică Photo/Video cu optimizare lumini și umbre. Opțional VR cu funcții interactive (deschidere uși/sertare). | Designeri, showroom-uri, prezentări clienți |
| **Kitchen Generator** | Modul specializat configurare bucătării din corpuri presetate cu randare și calcul cost automat. | Producători de bucătării |
| **Nesting** | Optimizare inteligentă tăiere panouri (ține cont de direcție fibră, grosime, tip material). Reduce risipa de material. | Fabrici cu producție în serie |

**Diferențiatori tehnici cheie:**
- **Proprietatea parametrică:** Modelele pot fi redimensionate și adaptate instantaneu. Productivitate cu până la 60% mai mare decât CAD tradițional la modificări de dimensiuni.
- **Integrare CNC nativă:** Workshop generează automat programare CNC direct din modelul 3D. Majoritatea competitorilor necesită software CAM separat (cost adițional + timp + erori de export).
- **Ecosistem complet:** Design + Workshop + Rendering într-o singură suită, date partajate nativ între module.
- **Made in Germany:** Origine germană = semnal puternic de calitate în piața românească de mobilier.
- **25+ ani pe piață:** Produs matur, stabil, cu track record demonstrat.

# PARTEA II: DIAGNOSTICUL STĂRII ACTUALE

## 6. Ce avem acum și ce nu funcționează

### 6.1 Site-ul actual (pytha3d.ro)
- **Problemă critică:** Este un SPA (Single Page Application) React care necesită JavaScript pentru a afișa conținutul
- **Consecință SEO:** Complet invizibil pentru Google. Crawlerele nu pot indexa conținutul. Zero trafic organic.
- **Consecință tracking:** Nu are Meta Pixel, nu are CAPI, nu are GA4, nu are Google Ads conversion tags. Campaniile rulează orb.
- **Textul indexat de Google:** Doar un singur snippet: "PYTHA unifică designul 3D, documentația tehnică și generarea codului CNC..."
- **Concluzie:** Site-ul trebuie reconstruit complet pe un stack SSR/SSG (Next.js)

### 6.2 Campaniile Meta actuale
- **Ce s-a făcut:** Campanii de Lead Generation cu formulare in-app Meta (F1, F2, F3)
- **Audientele testate:** AUDV1, AUDV2, AUDV3 cu multiple grafici (G1, G2)
- **Rezultat cu formulare simple (F1):** Lead-uri ieftine dar complet necalificate
- **Rezultat cu formulare complexe (F2, F3):** Lead-uri mai calificate dar cost per lead crescut substanțial
- **Problema fundamentală:** Lead-urile din formulare Meta NU trec prin nicio etapă de educație. Nu știu prețul, nu înțeleg modulele, nu au context. Când agentul sună și menționează 20.000 EUR, șocul prețului descalifică imediat o mare parte.

### 6.3 CRM-ul Excel (Leads_Central)
- **Total lead-uri în sistem:** 14
- **Status 0 (inițial/stagnant):** 9 (64%) — majoritatea nu au progresat
- **Status 1 (în progres):** 5 (36%)
- **Scor calitate medie:** 2.8/5 — sub medie
- **Lead-uri cu scor 0 (complet necalificat):** 2 (17%) — risipă de timp sales
- **Tip firmă predominant:** 87% "Small" — capacitate de plată discutabilă pentru 20.000 EUR
- **CNC:** 3 da, 2 nu, 1 "încă nu"
- **Software curent:** 40% nimic, 20% SketchUp
- **Date financiare completate:** 1 din 14 (lipsa datelor împiedică prioritizarea)
- **Sursă lead-uri:** 100% Meta Ads — dependență totală de un singur canal
- **Audientă dominantă:** V3 cu formular F2

### 6.4 Structura de etichetare MetaAds existentă
Sistemul de labeluri: **AD_AUD{versiune}_F{formular}_G{grafică}**
- AUD = Audiența (V1, V2, V3)
- F = Tipul formularului (F1 = date contact basic, F2 = F1 + tip firmă + CNC, F3 = F2 + când vrei demo + nr proiectanți)
- G = Grafică/Creative (G1, G2)
- Exemplu: AD_AUDV2_F2_G1 = Audiența V2, Formular cu tip firmă, Grafică 1

Acest sistem SE PĂSTREAZĂ și se integrează cu UTM-urile pe site (utm_content = label MetaAds).

### 6.5 Canale sociale existente
- Pagini Instagram și Facebook active
- Campanii de reclame pe Meta active
- Nu există Google Ads
- Nu există Mailchimp / email marketing
- Nu există strategie SEO / blog
- Nu există tracking integrat

## 10. Analiza funnel — unde se pierd banii

### Pierdere #1 — Lipsa etapei de educație (estimat: 40–60% din lead-uri pierdute)
Lead-urile din formulare Meta nu au trecut printr-un proces de informare. Nu știu prețul, nu înțeleg modulele. Când agentul sună și menționează 20.000 EUR, șocul prețului descalifică imediat o mare parte. Un site funcțional care afișează prețul transparent ar elimina această pierdere — cine completează formularul după ce a văzut prețul e serios.

### Pierdere #2 — Lipsa nurturing-ului (estimat: 20–30% din lead-uri pierdute)
Un lead care nu e gata de cumpărare azi nu înseamnă că nu va fi gata peste 3 luni. Fără email nurturing, aceste lead-uri se pierd complet. Mailchimp poate recupera o parte semnificativă.

### Pierdere #3 — Lipsa tracking-ului (impact: imposibilitatea optimizării)
Fără Meta Pixel pe site, fără CAPI, fără GA4, fără Google Ads conversions, algoritmii de advertising lucrează orbi. Nu pot optimiza către persoanele care într-adevăr convertesc. E ca și cum ai conduce noaptea cu farurile stinse.

# PARTEA V: INFRASTRUCTURA DE TRACKING

## 15. Google Tag Manager — ce este și cum funcționează

GTM este hub-ul central. Se instalează o singură dată pe site, apoi tot restul se configurează din interfața web. Trei concepte: **Tag-uri** (cod care se execută — Meta Pixel, Google Ads Conversion, GA4), **Triggere** (reguli: pe toate paginile, la submit formular, la vizitarea /preturi), **Variabile** (date dinamice: URL pagină, valori din DataLayer).

**DataLayer** = obiect JavaScript, "canal de comunicație" între site și GTM. Site-ul scrie în DataLayer (ex: `dataLayer.push({ event: 'demo_request' })`), GTM citește și declanșează tag-urile configurate.

**Container web vs server-side:** Web = rulează în browser (vulnerabil la ad blockers, iOS restrictions). Server-side = rulează pe server propriu (ocolește blocaje, first-party cookies). Recomandat: ambele.

### Tag-uri necesare în GTM:

|-----|---------|------|
| Google Tag (AW-xxx) | All Pages | Baza pentru Google Ads și GA4 |
| Conversion Linker | All Pages | Setează cookie _gcl_aw pentru atribuire |
| GA4 Configuration | All Pages | Tracking general site |
| Meta Pixel Base Code | All Pages | PageView automat |
| Meta Pixel — ViewContent | Pagini produs/modul | Tracking vizualizare conținut |

## 16. Meta Pixel + Conversions API (CAPI)

### Ce face Pixel-ul
Identifică vizitatorii de pe Meta, înregistrează acțiunile pe site, trimite date către Meta Events Manager. Meta folosește aceste date pentru optimizare campanii, retargeting, Lookalike audiences.

### Evenimente Meta necesare

| Eveniment | Când | Parametri |
|-----------|------|-----------|
| PageView | Fiecare pagină | Automat din Pixel |
| ViewContent | Pagini module, prețuri, case studies | content_name, content_category |
| Lead | Formular contact trimis | content_name: 'formular_oferta' |
| Schedule | Demo programat | content_name: 'demo_programat' |
| CompleteRegistration | Download broșură | content_name: 'brosura_pytha' |

### De ce CAPI e necesar
Pixel = browser-side (blocat de ad blockers, iOS ATT). CAPI = server-side (ocolește tot). Ambele trimit aceleași evenimente cu același event_id → Meta deduplică automat. Dacă Pixel e blocat, CAPI asigură datele. Event Match Quality (EMQ) ≥ 7 = trimite date hash-uite (email, telefon, nume) prin CAPI.

### Setup CAPI
1. Meta Events Manager → generează Access Token
2. Verifică domeniul pytha3d.ro (DNS TXT record sau meta tag)
3. Opțiunea A: GTM Server Container pe Stape.io (~10-50 EUR/lună) cu subdomeniu ss.pytha3d.ro
4. Opțiunea B: API routes Next.js direct cu Facebook Business SDK
5. Deduplicare: fiecare eveniment = același event_id în Pixel și CAPI
6. User data hash SHA-256 pentru EMQ ≥ 7

- **Secondary:** Formular Contact (Lead) — se măsoară dar nu optimizează licitarea
